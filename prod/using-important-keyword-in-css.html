Using the !important Keyword in CSS

Sure, it's able to override any rule, but what about its drawbacks?

If you've ever wielded the ultimate power in CSS, you know it's as easy as creating a selector and giving a property a value—only, at the end of the value, you're going to place !important before the semi-colon. Check the example below: 

ul {
  padding:0 1rem !important;
}

What !important allows you to do is bump this selector's property value specificity to the topmost order. This will override user agent styles, author styles, and computed styles. It will even override something that would otherwise be more specific, like this: 

.nav > .nav__container > ul {
  padding:0 2rem;
}

Gotchas

As the great adage goes: with great specificity comes great resposibility. (Or something like that.) Though it would be awesome to !important all your issues and walk away, there are things to be mindful of.

Shorthand

When you do something like this: 

.alert--primary {
  background: cornflowerblue !important;
}

What you're really doing is this: 

.alert--primary {
  background-url: initial !important;
  background-image: initial !important;
  background-clip: initial !important;
  background-position: initial !important;
  background-color: cornflowerblue !important;
  background-origin: initial !important;
  background-repeat: initial !important;
  background-size: initial !important;
  background-attachment: initial !important;
}

Now, if you're not familiar with how precedence works in CSS, this may seem like a double-whammy. Because we set the shorthand property background and only gave it a color (when really it accepts many others) this means that the other values were put back to their initial values.

As you can see, that's not all that happened here. When you declare a shorthand property as !important, you're effectively telling the browser to make all of its other properties important as well. And that kind of whoopsie will give us a major hit on...

Performance
