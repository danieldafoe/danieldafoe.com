Slimming Your CSS

Isn't it about time you put your stylesheets on a weight loss plan? Let's lessen the weight on users by cutting your selectors, which means improved speed, smaller file size, and happy users.

Before we get into the drastic measures for reducing your CSS size and increasing performance, let's take a quick look at what you should be doing already.

Specificity

What's great about this advice is that it's appropriate whether you're using vanilla CSS or something tastier like SASS. (God forbid you're using a post-processor. Are there meta CSS pre-processors yet?) Let's have a look-see at the first culprit:

.table tr > td .data { color:#fffffe; }

Seriously? This is awful. A great analogy for this is one's family tree. We don't often say "my brother's kid", do we? No. We say "my nephew". See, in every facet of life we try to minimize specificity!

If we look to the selector above we can ask ourselves an (!)important question: does the element with the .data class really appear somewhere else within a .table? If not, why are you trying to find one's only within a td element that is a direct child of a tr (which is also a problem in itself)?
